Are geolocation errors fixed in Sentinel-1 Collection 1?

Created in November 12, 2024

2024   ·   formatting   images   ·   sample-posts

The Sentinel-2 Level-1C Collection 1 represents the product dataset generated starting from processing baseline 05.XX included, info: Here

The new dataset has improved geolocation accuracy, however, how much it is better than old processing baselines?

This post compared the old baseline dataset (N0206) and new dataset (N0500) when generating datacube.

Processing to Level-2

All images were processed using Framework for Operational Radiometric Correction for Environmental monitoring (FORCE)

To address the geolocation uncertainty, FORCE has built-in Coregistration function

For the experiments, I created 4 datacube versions with FORCE:

  • Sentinel-2 images cube with baseline 0206, without co-registration
  • Sentinel-2 images cube with baseline 0206, with co-registration
  • Sentinel-2 images cube with baseline 0500, without co-registration
  • Sentinel-2 images cube with baseline 0500, with co-registration
